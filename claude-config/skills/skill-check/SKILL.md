"할 수 없어", "못 해", "불가능해" 같은 답변을 하기 전에 반드시 이 과정을 거칠 것.

## 실행 순서

1. **스킬 목록 확인**: 현재 로드된 스킬 목록을 확인
2. **도구 목록 확인**: 사용 가능한 도구(Bash, agent-browser, WebSearch, WebFetch 등) 재확인
3. **대안 탐색**: 직접적인 방법이 없어도 조합으로 가능한지 검토
   - agent-browser로 웹 UI 접근 가능한지
   - API/CLI 도구가 있는지
   - 서브 Claude 세션으로 위임 가능한지
   - 다른 기기(회사컴 SSH 등)에서 가능한지
4. **실행 또는 솔직한 설명**: 방법을 찾으면 바로 실행. 정말 없으면 "왜" 못하는지 구체적으로 설명하고 대안 제시

## 핵심 원칙

- "못 해"라고 바로 답하지 말 것
- 최소 30초는 방법을 탐색할 것
- 조합/우회 방법도 고려할 것
- 정말 불가능하면 이유와 대안을 같이 제시

## 자동 트리거

이 스킬은 명시적 호출 없이도, "할 수 없다"는 판단이 들 때 자동으로 내부적으로 실행할 것.
